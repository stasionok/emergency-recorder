Android · 8.0+

Что делает приложение

Emergency Recorder — это инструмент для быстрого включения аудиозаписи (например, по виджету или жесту) в ситуациях, когда важна каждая секунда. Запись сохраняется на устройстве и одновременно стримится в облачные хранилища, которые подключил сам пользователь — чтобы файл остался доступен, даже если устройство будет потеряно или повреждено.

  • Запуск записи через виджет домашнего экрана, плитку быстрых настроек или Accessibility-сервис
  • Параллельный стриминг во все подключённые облачные аккаунты
  • Автоматический повтор загрузки при потере связи
  • Локальная копия каждой записи на устройстве
  • Шифрование настроек и токенов на устройстве (Argon2id + AES-GCM)

Поддерживаемые облачные сервисы

Пользователь сам выбирает, куда загружать свои записи. Никаких файлов не отправляется на сервера разработчика — только в аккаунты, к которым пользователь дал доступ напрямую.

  • Google Drive (через нативный Account Picker, scope drive.file)
  • Dropbox (OAuth 2.0 + PKCE)
  • Yandex.Disk (OAuth 2.0)
  • pCloud (OAuth 2.0 + PKCE)
  • Box (OAuth 2.0 + PKCE)
  • Mail.ru Cloud (WebDAV + app-specific password)
  • Любой WebDAV-сервер
  • SFTP-сервер
  • S3-совместимое хранилище

Какие данные собирает приложение

  • Аудиозаписи, которые пользователь явно создаёт через приложение. Хранятся локально на устройстве пользователя.
  • OAuth-токены для облачных сервисов, к которым пользователь подключился. Хранятся только на устройстве пользователя в зашифрованном виде (Argon2id для деривации ключа + Tink AES-GCM).
  • Настройки приложения — выбор активных облаков, имя папки и т.п. Хранятся только на устройстве.

Куда отправляются данные

  • Аудиозаписи отправляются только в те облачные аккаунты, которые пользователь сам подключил, и только напрямую — по официальным API провайдеров (Google Drive, Dropbox, Yandex.Disk, pCloud, Box, Mail.ru, WebDAV/SFTP/S3-серверам пользователя).
  • Разработчик не имеет собственных серверов, на которые поступали бы записи или метаданные пользователей.
  • Приложение не использует сторонние трекеры аналитики и не отправляет crash-логи третьим сторонам.

OAuth scope

Приложение запрашивает минимально необходимые scope у каждого провайдера:

  • Google Drive: https://www.googleapis.com/auth/drive.file — доступ только к файлам, которые создало само приложение.
  • Dropbox: files.content.write files.content.read с offline access.
  • Yandex.Disk: cloud_api:disk.write cloud_api:disk.read cloud_api:disk.info.
  • pCloud / Box: запись и чтение в выделенной папке приложения.

Удаление данных

Пользователь может в любой момент:

  • Удалить локальные записи через файловый менеджер устройства или внутри приложения.
  • Отключить любой облачный аккаунт в настройках приложения — токены немедленно удаляются с устройства.
  • Отозвать доступ приложения через панель управления соответствующего облачного провайдера.
  • Удалить приложение — все локальные данные и токены удаляются вместе с ним.

Дети

Приложение не предназначено для детей младше 13 лет и не собирает целенаправленно данные от них.
