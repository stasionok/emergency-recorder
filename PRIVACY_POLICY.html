1. Кто мы

Приложение Emergency Recorder (applicationId: ru.stasionok.emergencyrecorder) разработано физическим лицом — далее «Разработчик». Контактный адрес: stasionok@gmail.com.

2. Какие данные собираются

2.1. Аудиозаписи

Приложение записывает звук по явной команде пользователя и сохраняет файл в формате AAC (ADTS) во внутреннее хранилище устройства (app-private storage). Разработчик не имеет доступа к содержимому записей и не хранит аудиофайлы на своих серверах.

Если пользователь настроил облачное хранилище (Google Drive, Dropbox, Yandex.Disk, pCloud, Box, Mail.ru, WebDAV, SFTP, Amazon S3), запись передаётся напрямую в это хранилище по зашифрованному соединению (HTTPS/TLS). Целевое хранилище — собственный аккаунт пользователя, к которому Разработчик не имеет доступа.

2.1.1. OAuth-доступ к облакам

Для подключения к Google Drive, Dropbox, Yandex.Disk, pCloud и Box используется стандартный протокол OAuth 2.0. Пользователь входит в свой аккаунт на странице самого провайдера (в браузере, через Custom Tabs); приложение получает только access-токен и refresh-токен с минимально необходимыми правами:

  • Google Drive: drive.file — доступ только к файлам, созданным самим приложением;
  • Dropbox: files.content.write files.content.read с offline access;
  • Yandex.Disk: cloud_api:disk.write disk.read disk.info;
  • pCloud / Box: доступ к выделенной папке приложения.

Логин и пароль пользователя никогда не попадают в приложение и не передаются Разработчику. Полученные токены хранятся только на устройстве пользователя в зашифрованном виде. Для Mail.ru используется WebDAV с app-specific password (создаётся пользователем в настройках Mail.ru), для SFTP/WebDAV/S3 — учётные данные, введённые пользователем вручную (также хранятся только на устройстве в зашифрованном виде).

2.2. Идентификатор устройства (хеш)

Для верификации подписки приложение использует одностороннее хеш-значение, производное от Android Installation ID. IMEI, рекламный ID (GAID) и серийный номер не используются. Хеш передаётся на серверы Google Play / RuStore исключительно в целях проверки статуса подписки.

2.3. Адрес электронной почты

Email-адрес собирается опционально при оформлении подписки, для отправки служебных уведомлений о статусе подписки: успешное продление, ошибка списания, истечение. Пользователь может отказаться от получения email-уведомлений при оформлении подписки — премиум-функции при этом не затрагиваются.

4. Используемые сторонние сервисы

  • Google Play Billing — обработка платежей и подписок. Политика Google: policies.google.com/privacy
  • RuStore Billing (только в RuStore-сборке) — обработка платежей. Политика VK: rustore.ru/help/legal
  • Google Play Developer API — проверка статуса подписки на стороне сервера (только Purchase Token, без персональных данных).
  • Resend (resend.com) — сервис для отправки служебных email-уведомлений. Передаются только email-адрес и текст уведомления. Политика Resend: resend.com/legal/privacy-policy
  • Google Cloud Pub/Sub — получаем уведомления от Google Play о статусе подписок (Real-time Developer Notifications). Сами в Pub/Sub ничего не публикуем — это односторонная входящая очередь сообщений.

4а. Сроки хранения

  • Email-лог (таблица `email_log`): 90 дней.
  • Subscription tokens (токены покупок): до окончания подписки + 30 дней, затем автоматически удаляются.
  • Аудиозаписи: хранятся локально на устройстве или в облаке пользователя до явного удаления.

5. Специальные возможности (Accessibility Service)

Приложение использует Accessibility Service исключительно для обнаружения комбинаций физических кнопок громкости при заблокированном экране. Сервис не считывает, не передаёт и не анализирует содержимое других приложений или экрана. Единственное действие, которое он выполняет — обнаружение нажатия кнопки и передача сигнала сервису записи.

6. Где хранятся данные

  • Аудиозаписи — на устройстве пользователя и/или в его личном облачном хранилище.
  • Настройки и хеш-ID — на устройстве пользователя.
  • Серверы Разработчика — используются только для верификации подписки (хранится только Purchase Token и его статус, без аудио и персональных данных).

1. Who We Are

The Emergency Recorder application (applicationId: ru.stasionok.emergencyrecorder) is developed by an individual developer ("Developer"). Contact: stasionok@gmail.com.

2. Data We Collect

2.1. Audio Recordings

The app records audio on explicit user command and saves the file in AAC (ADTS) format to the device's private internal storage. The Developer cannot access recording content and does not store audio files on any servers.

If the user configures a cloud storage target (Google Drive, Dropbox, Yandex.Disk, pCloud, Box, Mail.ru, WebDAV, SFTP, Amazon S3), the recording is transmitted directly to that storage over an encrypted connection (HTTPS/TLS). The destination is the user's own storage account — the Developer has no access to it.

2.1.1. OAuth Access to Clouds

Connections to Google Drive, Dropbox, Yandex.Disk, pCloud, and Box use the standard OAuth 2.0 protocol. The user signs in on the provider's own page (in-browser via Custom Tabs); the app receives only an access token and a refresh token with minimum required scopes:

  • Google Drive: drive.file — access only to files the app itself creates;
  • Dropbox: files.content.write files.content.read with offline access;
  • Yandex.Disk: cloud_api:disk.write disk.read disk.info;
  • pCloud / Box: access to the app's dedicated folder.

The user's login and password never reach the app and are not transmitted to the Developer. Tokens received are stored only on the user's device, encrypted. For Mail.ru, a WebDAV connection with an app-specific password (created by the user in Mail.ru settings) is used; for SFTP/WebDAV/S3, credentials are entered by the user manually (also stored only on the device, encrypted).

2.2. Device Identifier (Hash)

For subscription verification, the app uses a one-way hash derived from the Android Installation ID. IMEI, Advertising ID (GAID), and device serial number are not used. The hash is submitted to Google Play / RuStore servers solely to verify subscription status.

2.3. Email Address

An email address is collected optionally when subscribing, to send transactional notifications about subscription status: successful renewal, payment failure, expiration. The user may opt out of email notifications at subscription checkout — premium features are unaffected by this choice.

5. Accessibility Service

The app uses Android's Accessibility Service solely to detect physical volume button combinations when the screen is locked. The service does not read, capture, or transmit content from other apps or the screen. Its only action is detecting a button event and signaling the recording service.

6. Where Data Is Stored

  • Audio recordings — on the user's device and/or in the user's personal cloud storage.
  • Settings and hashed device ID — on the user's device.
  • Developer's servers — used only for subscription verification (stores Purchase Token and its status; no audio, no personal data).
